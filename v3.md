# VisProbe v3.0 — Implementation Plan

**Target release:** v3.0.0 (single release; folds the internal-refactor changes from the post-v2.0 cleanup *and* the new RobustBench integration into one version bump).

**Scope:** ~1100 new lines of code, ~18-19 days of focused engineering, ~5-6 weeks real-clock with `robustbench` API drift contingency.

**Two user-facing features behind one shared safety mechanism:**

- **Feature A — Official rank.** Run a strict-protocol eval, get a defensible rank against the published RobustBench leaderboard.
- **Feature B — Head-to-head on your data.** Download the top-k published robust models, re-evaluate them on *your* images under *your* compositional protocol, and rank yourself alongside.

The shared safety mechanism is **protocol gating**: a metadata-check function (`validate_protocol`) that refuses to return a leaderboard rank unless the underlying evaluation matches RobustBench's protocol exactly. Bugs in this function cause exactly the harm the feature exists to prevent. It gets disproportionate test coverage.

## Cross-cutting concerns (applied to every milestone)

### Cost warnings (`confirm=True` pattern)

Every expensive operation prints an honest cost estimate before doing any compute, and requires `confirm=True` to actually run. This makes the right choice (strict protocol) the obvious one — users see the cost upfront instead of cargo-culting APGD-CE and then tripping the gate.

```python
def robustbench_eval(model, dataset, threat, *, confirm=False):
    cost = _COST[(dataset, threat)]
    msg = (
        f"RobustBench protocol on {dataset}/{threat}:\n"
        f"  {cost['n_samples']} samples, full AutoAttack-standard\n"
        f"  estimated: {cost['hours_a100']}h on A100 (~${cost['usd_a100']})\n"
    )
    if not confirm:
        raise RuntimeError(msg + "\nPass confirm=True to proceed.")
    print(msg)
    # ... actual eval
```

Applies to `robustbench_eval()` and `compare_against_top_k()`. Documented in README and the function docstrings.

### Schema versioning in snapshots

Every snapshot JSON carries `"schema_version": 1` at the top level. `RobustBenchClient.__init__` raises on `schema_version > _MAX_KNOWN_SCHEMA`. Costs nothing now, saves a migration painpoint when RobustBench adds new fields.

### Float tolerance is symmetric

Every check that could see floating-point drift uses `abs(x) > 1e-9`, not `x != 0.0`. Applies to `severity`, `eps`, and any future scalar protocol field.

### M3 — Protocol gate  *(1-2 days)*

**Goal:** The single function that makes the whole feature honest. Maximum test coverage.

**Deliverables:**
- `src/visprobe/leaderboard.py` (initial ~80 LOC):
  - `_PROTOCOL` spec table: `{(dataset, threat): {n_samples, eps, attack}}`
  - `class ProtocolError(Exception)`
  - `def validate_protocol(result: EvaluationResult, dataset: str, threat: str) -> None`
- `tests/test_leaderboard.py` (initial ~80 LOC):
  - One test per protocol-violation case: missing protocol tag, wrong n_samples, wrong eps, wrong attack, severity != 0
  - One happy-path test
  - Test float tolerance: `severity=1e-12` passes, `severity=1e-7` fails

**Acceptance:**
- `pytest tests/test_leaderboard.py -k protocol` — all pass
- **Error-message contract is tested, not vibes-checked.** Each violation type has an explicit assertion on the substring users will see, e.g.:
  - missing protocol tag: `assert "use robustbench_eval()" in str(exc)`
  - wrong n_samples: `assert "n_samples=" in str(exc) and "expected " in str(exc)`
  - wrong eps: `assert "eps=" in str(exc)`
  - wrong attack: `assert "autoattack-standard" in str(exc)` (the expected value)
  - env perturbation: `assert "severity=" in str(exc)`
- The protocol gate is the user's first impression of the safety mechanism; pin the message contract in tests so it can't silently regress.

### M4 — `RobustBenchClient`  *(1-2 days)*

**Goal:** Snapshot reader with ranking logic.

**Deliverables:**
- `leaderboard.py` extended with `class RobustBenchClient`:
  - `__init__(dataset, threat)` — loads snapshot via `importlib.resources`, raises if `schema_version > _MAX_KNOWN_SCHEMA`
  - **Validates each entry on load** (~10 LOC): each entry has `name: str`, `rank: int`, `clean_acc: float in [0,1]`, `robust_acc: float in [0,1]`, no `None` values for required fields. Catches refresher bugs (e.g., a model with `robust_acc: null`) before they cause downstream confusion in ranking math.
  - `rank(robust_acc: float) -> int` — where this number lands in the sorted leaderboard
  - `top_k(k: int) -> list[dict]` — top-k entries
  - `neighbors(robust_acc, k=3) -> tuple[list, list]` — (above, below)
  - `snapshot_date: str` property
  - `__len__()` returns entry count
- Tests for rank math (boundary cases: top, bottom, ties), top_k, neighbors, schema version mismatch, **and entry validation** (corrupt entry → `ValueError` with the bad field named)

**Acceptance:**
- All rank-math tests pass including: tied robust_acc, lower than all entries, higher than top
- Snapshot date is accessible
- Schema bump simulation: loading a `schema_version: 99` snapshot raises a clear error
- Corrupt-entry simulation: a snapshot with `robust_acc: null` or missing `name` field raises with the bad field identified

### M6 — `robustbench_eval()` helper  *(2 days)*

**Goal:** One-line entrypoint that runs a fully-pinned RobustBench-protocol eval. Cost-warned. Optional-dep gated.

**Deliverables:**
- `src/visprobe/experiment.py` extended (~95 LOC, standalone function not method):
  - `def robustbench_eval(model, dataset, threat, *, batch_size=50, device="cuda", confirm=False) -> EvaluationResult`
  - Lazy-imports `robustbench.data.load_cifar10` / `load_imagenet`
  - Raises `ImportError` with install instructions if `robustbench` not present
  - Prints cost estimate; requires `confirm=True` to proceed
  - Constructs `CompositionalExperiment` with pinned params:
    - `attack="autoattack-standard"`, exact eps from `_PROTOCOL`, `severities=[0.0]`
    - `env_strategies={"none": lambda images, severity: images}` — **identity callable, NOT an empty dict** (empty dict means zero loop iterations and silently produces no results; the identity strategy makes the eval run with no perturbation while still going through the loop)
  - Tags resulting `EvaluationResult.metadata` with `{"protocol": "robustbench", "dataset": ..., "threat": ...}`
  - Returns the single `EvaluationResult`
- `pyproject.toml`: add `[robustbench]` optional extra (`"robustbench>=1.1"`)
- Tests with `unittest.mock` for `robustbench` module:
  - Confirm-not-passed raises with cost message
  - Metadata correctly tagged
  - Missing `robustbench` package raises with install hint

**Acceptance:**
- Calling without `confirm=True` raises with the full cost estimate visible
- Resulting `EvaluationResult` passes `validate_protocol()` for its (dataset, threat)
- Without `robustbench` installed, helpful `ImportError` mentions `pip install visprobe[robustbench]`

## Risks (unchanged from earlier discussion)

- **`robustbench` package API drift.** Their `load_model` signature has shifted historically. Pin to a known-good version range; isolate via `model_zoo.py` so drift contained to one file. Allocate ~1 day for drift contingency.
- **Head-to-head GPU cost.** Re-evaluating 5 ImageNet robust models with full AutoAttack on a 1000-image set is hours of GPU time. Default `k=5` but cost-warn and allow `k=3` or specific-name selection.
- **User confusion between Feature A and Feature B.** Mitigated by: separate methods, separate dataclasses, separate render styles, mandatory `data_source: "user"` flag on every Feature B output.
- **Protocol gate bugs.** This is the safety mechanism — bugs here cause exactly the harm the feature is designed to prevent. Maximum test coverage; review carefully.

## What gets built first

**M3, the protocol gate.** Before any snapshot, before any model zoo. If `validate_protocol()` is solid with its full test matrix, everything else hangs cleanly off it. If it's wrong, every downstream feature is unsafe to ship regardless of polish.

Build it standalone (using hand-stubbed `EvaluationResult` instances in tests, no dependency on snapshots) so it can land and be reviewed before M1 commits any data files.
